# Лабораторная работа №1 по методам оптимизации

## Цель

Исследование и сравнение методов одномерной оптимизации на отрезке в графическом приложении.

## Что реализовано

- поиск `min/max` на отрезке;
- методы: дихотомия, золотое сечение, Фибоначчи;
- поддержка двух шаблонов функции:
  - `f(x) = a·x² + b·x + c`;
  - `f(x) = (a·x² + b·x + c) / (d·x² + e·x + f)`;
- единый запуск с поддержкой одного или нескольких значений `ε` и `l` (все комбинации `ε × l`);
- таблица итераций и график функции с найденными точками.

## Структура пакета

- `lr1/domain/` — доменные модели, шаблоны функций, численные методы.
- `lr1/application/` — валидация входа, use-case сценарии, подготовка view-model.
- `lr1/ui/` — окно, вкладки, фоновые задачи, построение графиков.
- `lr1/infrastructure/` — конфигурация и логирование.
- `lr1/main.py`, `lr1/__main__.py` — точки входа.

## Требования

- Python `3.10+`.
- `pip`.
- зависимости из общего файла [`requirements.txt`](../requirements.txt).

## Установка

```bash
cd optimmethods
python3 -m venv .venv
source .venv/bin/activate
python -m pip install -r requirements.txt
```

## Запуск

```bash
cd optimmethods
source .venv/bin/activate
python -m lr1
```

## Использование

1. В блоке `Функция` выбрать шаблон и задать коэффициенты.
2. В блоке `Сценарий` выбрать цель (`Максимум`/`Минимум`) и метод.
3. В блоке `Параметры` задать `a`, `b`, а значения `ε` и `l` добавить кнопками `+` (лишние удалить `−`).
4. Нажать `Рассчитать` — приложение автоматически выполнит все комбинации `ε × l`.
5. Анализировать вкладку `Итоги и итерации` и вкладку `Графики`.

## Ограничения и валидация

- должно выполняться `a < b`;
- `ε > 0`, `l > 0`;
- для дихотомии: `ε < l`;
- интервал не должен пересекать точки разрыва рациональной функции;
- тождественный ноль знаменателя запрещен.
