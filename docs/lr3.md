# Лабораторная работа №3 по методам оптимизации

## Цель

Поиск минимума или максимума функции двух переменных градиентными методами в графическом приложении.

## Что реализовано

- градиентный метод первого порядка с выбором `min/max`;
- метод сопряженных градиентов (Fletcher-Reeves) с выбором `min/max`;
- безопасный парсинг выражения `F(x1, x2)` с whitelist-операциями и без небезопасного `eval`;
- явный выбор цели `min/max` в UI и в конфиге запуска;
- журнал итераций с причиной остановки;
- общий аналитический блок: градиент, стационарная точка, матрица Гессе и вывод о характере экстремума;
- метрики запуска (`trace_id`, `latency`);
- графики траектории на линиях уровня и сходимости значения функции с привязкой к цели;
- единый Qt-интерфейс в том же стиле, что ЛР1 и ЛР2, включая embedded-режим в shell.

## Структура пакета

- `lr3/domain/` — контракты, компилятор выражений, численные методы.
- `lr3/application/` — парсинг пользовательского ввода и запуск методов.
- `lr3/ui/` — GUI на `PySide6` и визуализация результатов.
- `lr3/tests/` — тесты ЛР3 (алгоритм + архитектурные границы).
- `lr3/main.py`, `lr3/__main__.py` — точки входа.

## Требования

- Python `3.10+`.
- `pip`.
- зависимости из общего [`requirements.txt`](../requirements.txt).

## Установка

```bash
cd optimmethods
python3 -m venv .venv
source .venv/bin/activate
python -m pip install -r requirements.txt
```

## Запуск

```bash
cd optimmethods
source .venv/bin/activate
python -m lr3
```

## Использование

1. Выбрать метод (`Градиентный` или `Сопряженные градиенты`).
2. Ввести функцию `F(x1, x2)` или оставить дефолт для выбранного метода.
3. Выбрать цель: `Поиск минимума` или `Поиск максимума`.
4. Задать стартовую точку и параметры (`epsilon`, `initial_step`).
5. Внутренние ограничения шага и числа итераций задаются автоматически.
6. Нажать `Рассчитать` и анализировать учебный отчёт: сначала теория экстремума, затем итерации и графики.

## Ограничения и валидация

- в выражении разрешены только `x1`, `x2`, `pi`, `e`, `sin`, `cos`, `tan`, `exp`, `log`, `sqrt`, `abs`, `pow`;
- по достижении `timeout_seconds` выполнение завершается с причиной `timeout`;
- пороги и лимиты задаются через `MethodConfig`.

## Migration note (breaking change)

- UI ЛР3 переведен с `Tkinter` на `PySide6` для унификации с ЛР1/ЛР2 и shell;
- ЛР3 теперь поддерживает embedded-режим в общем приложении лабораторных;
- отдельный запуск `python -m lr3` сохранен.
