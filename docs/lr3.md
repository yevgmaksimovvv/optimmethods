# Лабораторная работа №3 по методам оптимизации

ЛР3 оформлена отдельным пакетом `lr3` и повторяет архитектурный шаблон `lr1/lr2`: контракт домена, прикладной слой и UI.

## Что реализовано

- градиентный подъём для поиска максимума;
- метод сопряжённых градиентов (Fletcher-Reeves);
- безопасный парсинг пользовательской функции `F(x1, x2)` без небезопасного `eval` по произвольному коду;
- таблица итераций, причина остановки, метрики запуска (`trace_id`, latency);
- графики: траектория на линиях уровня и сходимость значения функции.

## Структура ЛР3

- `lr3/__main__.py` — запуск через `python -m lr3`.
- `lr3/main.py` — точка входа и проверка зависимостей GUI.
- `lr3/domain/models.py` — доменные контракты (`MethodConfig`, `OptimizationResult`).
- `lr3/domain/expression.py` — безопасный компилятор выражений.
- `lr3/domain/methods.py` — численные методы и логика остановки.
- `lr3/application/services.py` — валидация пользовательского ввода и оркестрация запуска.
- `lr3/ui/window.py` — GUI и визуализация результатов.

## Запуск

```bash
source .venv/bin/activate
python -m lr3
```

## Ограничения

- выражение может использовать только `x1`, `x2`, `pi`, `e`, функции: `sin`, `cos`, `tan`, `exp`, `log`, `sqrt`, `abs`, `pow`;
- при достижении `timeout_seconds` метод завершается с причиной `timeout`;
- все пороги остановки и лимиты задаются через `MethodConfig`.
