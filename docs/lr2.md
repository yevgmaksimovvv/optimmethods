# Лабораторная работа №2 по методам оптимизации

ЛР2 выделена в отдельный пакет `lr2` и должна повторять общий архитектурный шаблон проекта: `domain` + `application` + `ui` + локальные тесты.

## Текущее состояние

На момент фиксации этого документа исходники ЛР2 отсутствуют в рабочем дереве ветки (сохранились только служебные директории и кэш-артефакты). Это согласуется с известной проблемой ветки.

## Целевая структура ЛР2

- `lr2/domain/` — доменный слой и контракты.
- `lr2/application/` — прикладные use-case сценарии.
- `lr2/ui/` — графический интерфейс.
- `lr2/tests/` — тесты только для ЛР2.

## Правила для ЛР2

- зависимости берутся из общего `requirements.txt` в корне;
- отдельные `requirements` внутри `lr2/` не допускаются;
- общие тесты в корне репозитория не используются;
- при восстановлении кода ЛР2 документация обновляется по фактической реализации.
