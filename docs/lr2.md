# Лабораторная работа №2 по методам оптимизации

## Цель

Поиск минимума функции нескольких переменных методом Розенброка (непрерывный шаг) через графический интерфейс.

## Что реализовано

- целевая функция задается полиномом двух переменных через матрицу коэффициентов `c[i][j]`;
- поддерживаются пресеты функций из задания;
- метод Розенброка запускается для набора `ε` и набора стартовых точек;
- выводится таблица итераций;
- доступны 2D-контуры и 3D-поверхность с траекторией;
- доменный solver поддерживает `N`-мерный контракт, GUI работает как 2D-адаптер.

## Структура пакета

- `lr2/domain/` — модели, полиномиальная функция, метод Розенброка.
- `lr2/application/` — конфигурирование и orchestration запусков.
- `lr2/ui/` — GUI и визуализация результатов.
- `lr2/tests/` — тесты ЛР2 (алгоритм + архитектурные границы).
- `lr2/main.py`, `lr2/__main__.py` — точки входа.

## Требования

- Python `3.10+`.
- `pip`.
- основной путь установки: зависимости из общего [`requirements.txt`](../requirements.txt).

## Установка

```bash
cd optimmethods
python3 -m venv .venv
source .venv/bin/activate
python -m pip install -r requirements.txt
```

## Запуск

```bash
cd optimmethods
source .venv/bin/activate
python -m lr2
```

## Использование

1. Задать коэффициенты полинома `c[i][j]` или выбрать пресет.
2. Указать один или несколько `ε`.
3. Указать одну или несколько стартовых точек `(x1, x2)`.
4. Запустить расчет и выбрать нужный запуск в таблице результатов.
5. Переключать визуализацию: `Контуры 2D` / `3D поверхность`.

## Ограничения и валидация

- текущий UI ограничен 2D-вводом (`x1`, `x2`);
- пустая ячейка коэффициентов трактуется как `0`;
- для вырожденных случаев включены критерии остановки по стагнации и fallback перестройки направлений.

## Migration note (breaking change)

- убран выбор размерности таблицы коэффициентов, закреплен контракт `c[0..4][0..4]`;
- solver работает через контракт `objective(vector)` и `N`-мерную стартовую точку;
- для line-search используется явный диапазон `lambda` из конфига.
